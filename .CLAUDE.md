# Element Hunter - Claude Code メモ

## クローラー起動方法

```bash
# tools/crawler ディレクトリから実行
cd c:/home/project/element-hunter/tools/crawler
node dist/index.js <URL> [オプション]

# 例: agile-studio.jp をクロール（サイトID指定）
node dist/index.js https://agile-studio.jp -i agile-studio -v

# 例: esm.co.jp をクロール
node dist/index.js https://esm.co.jp -i esm -v
```

### オプション

| オプション | 説明 | デフォルト |
|-----------|------|-----------|
| `-i, --site-id <id>` | サイトID（ファイル名） | ホスト名から自動生成 |
| `-n, --site-name <name>` | サイト名 | ページタイトルから取得 |
| `-d, --max-depth <n>` | 最大クロール深度 | 3 |
| `-p, --max-pages <n>` | 最大ページ数 | 50 |
| `-v, --verbose` | 詳細ログ出力 | false |
| `-o, --output <dir>` | 出力ディレクトリ | `../../data/sites` |

### ビルドが必要な場合

```bash
cd c:/home/project/element-hunter/tools/crawler
npm run build
```

## 開発サーバー

```bash
cd c:/home/project/element-hunter
npm run dev
```

## Viteキャッシュクリア

サイト一覧が正しく反映されない場合:

```bash
rm -rf c:/home/project/element-hunter/node_modules/.vite
```

その後、開発サーバー再起動 + ブラウザでCtrl+Shift+R（ハードリロード）
